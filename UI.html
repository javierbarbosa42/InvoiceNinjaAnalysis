Given the nature of the application (business) and the serious tone it needs to have, a simple color palette is probably preferable rather than a colorfoul and less serious color scheme


Let's take a look at the font choice for the application. This and every other screen has as font choice Roboto. Which is a very common choice, given it's simplistic and straight to the point design

Roboto has it's own origin and meaning.

  • Origin: Roboto was created by a desginer at google Crhistian Robertson. It was developed with the special intent of it being used on Android devices for digital usage. It has since then been categorized as the default Android font.
  • Meaning: Google itself ha described Roboto as modern, yet approachable with a touch of emotion.

As said previously, Roboto has a clean and straightforward design that is pleasant and familiar to look at. It's a good choice considering the nature a purpouse of the application


Aesthetically the app is not pretty, nevertheless is not ugly to look at. Looking at the app is not very apparent that a tremendous amount of work has been put into it and that it has over 2000 commits with more than 10,000 lines of code. Even so, it is not an ugly looking app, it's simplicity fulfills it's purpouse. It is not a dense or overloaded screen. Everything looks organized & structured.

Nevertheless, it is not a very creative or interesting design. Yes, it's simplicity fulfills it's purpouse, and given the sensitive information g handled in this application is probably best to have a simple design, but the app doesn't have the company touch that should characterize it as an application developed by invoice ninja and not any person out there.

    Settings screen:

    The settings screen, shown above. Shares some of the components of the other screens. But most of them are not in this screen. The settings screen is one of the most minimalistic screens of the application. It only has for 3 different user choices

    • Dark mode
    • Biometric Authentication
    • Auto Start Tasks

    It also has the option to refresh data. Why refresh?

    Refresh is an option offered in this application given it's nature. As explained in the general description section, this application has a both mobile and web sides. In the web side the users are able to do just as much as in the mobile version. Therefore users are able to create invoices, clients, products, etc... in the web application. But it would be really useless if what you did on the web app did not transfer to the mobile version. This is what the refresh button is for. Refresh mobile version and update anything that is not up to date with the web version.
