Summary & procedure

In this report Invoice Ninja is analized. Invoice ninja is a mobile and web application developed in flutter. This report is done by Javier Barbosa and David Niño for 'construcción de aplicaciones móviles'

The application version studied in this report corresponds to the version: 0.1.53

The application is available in the play sotre through the following link: https://play.google.com/store/apps/details?id=com.invoiceninja.flutter&hl=en

It is also availabe on the App Store through the following link: https://apps.apple.com/us/app/invoice-ninja/id1435514417?ls=1

For this report, the application version that will be studied is the Android version

    Objectives
  • Describe and analyze the application features
  • Find similar applications in the market
  • Describe the differentiating factors of other applications with this application
  • Describe and analize the UI of the application
  • Examin the UX of the application
  • Observe and analize eventual connectivity features of the application
  • Examine the design choices made in the building of the application
  • Perform and analize perfomance studies on the applications
    Strategies
  • The mobile application will be studied, using it on a mobile device, subjecting it to different quality scenarios
  • Application information will be collected in terms of performance, profiling and tracking of standards in the code using external software (such as Firebase, Android Studio, Codacy and MobSF).
  • The project will be analyzed understanding the characteristics of the source code.

The analysis of the application will be developed in a series of stages:

  1. General Description of the application and the repo
  2. View study
  3. Performance analisys
  4. Code inspection
  5. Bug analisys
  6. Architecture analisys
