Bug 3: Login fails (iOS 13.1.2)


This bug is impeding the app to keep running when the PDF viewer is opened.

As described by the user that issued this bug: On iOS the login screen (self hosted) fails. If I can manage to fill out the credentials (see below) then tapping login empties all fields. As far as I can see from the access logs on my server no connection attempt is made. Password manager integration only works 50% of the time to fill out the fields. The rest of the time you can select a login but nothing happens. Additionally switching away from the app (such as to your password manager) and back empties all the credential fields, after a period of a few minutes this might be wise from a security perspective but inside of a minute it is an exercise in frustration.
